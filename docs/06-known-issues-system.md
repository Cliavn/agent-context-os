# 历史问题系统

## 目标

历史问题系统用于把 Bug 修复经验沉淀成可复用的防线，避免同类问题反复出现。

## 问题记录结构

```md
# 问题：<标题>

状态：open / fixed / monitoring / archived
首次发现日期：
影响模块：
关联代码：
关联测试：

## 现象

## 期望行为

## 根因

## 修复方案

## 防复发规则

## 回归验证

## 关联任务
```

## 防复发机制

Bug 修复后至少选择一种沉淀方式：

- 新增或更新回归测试。
- 更新 checklist。
- 更新模块卡片的常见风险。
- 更新业务规则或状态机。
- 更新质量门禁。

## 问题优先级

| 优先级 | 说明 |
|---|---|
| P0 | 会导致数据错误、权限问题、安全问题或主流程不可用 |
| P1 | 会导致关键功能失败或高频误操作 |
| P2 | 影响局部体验或低频场景 |
| P3 | 文案、样式或低风险边缘问题 |
