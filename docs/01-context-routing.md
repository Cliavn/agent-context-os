# 上下文路由

## 目标

上下文路由的目标是让 Agent 用最少 Token 读取最有用的信息。新版路由不再默认从用户项目读取完整 `docs/agent/`，而是先读取极薄入口，再通过项目记忆源和本地检索索引定位真实上下文。

## 分层加载策略

Agent 应按以下顺序加载上下文：

| 层级 | 内容 | 读取条件 |
|---|---|---|
| L0 | 项目 `AGENTS.md` | 每次任务必读 |
| L1 | `.agent-context/config.json` | 每次任务必读 |
| L2 | 当前协作引擎规则 | 按 `engine.source` 加载，确定任务拆解、S0-S3 和门禁 |
| L3 | 项目记忆源摘要 | 涉及业务、交互、实现习惯、历史坑或模块边界时检索 |
| L4 | 本地检索索引 | 已存在且校验通过时优先使用；缺失或过期时重建 |
| L5 | 真实代码、测试或项目文档 | 根据检索命中的路径读取 |
| L6 | 方案台账或决策记录 | 用户要求定版、任务清单或完整方案落地时读取 |
| L7 | 历史文档 | 只有冲突、回溯或复发问题需要时读取 |

## 任务路由示例

| 任务类型 | 必读上下文 |
|---|---|
| 新项目接入 | `AGENTS.md`、`.agent-context/config.json`、项目检查脚本 |
| 老项目原地换芯 | 旧协作引擎文件清单、有效项目记忆、极薄入口模板 |
| S0 微小变更 | 入口、当前引擎 S0 规则、相关代码或样式文件 |
| 新增功能 | 当前引擎新功能 workflow、相关项目记忆源、代码证据、验证命令 |
| 修复 Bug | 当前引擎 bug workflow、历史坑记忆、相关代码和回归测试 |
| UI 改造 | 交互习惯和端侧风格记忆、相关页面代码 |
| 字段或状态变更 | 业务规则记忆、字段语义、状态流转、测试 |
| 完整方案开发 | 当前引擎方案台账规则、confirmed / active 台账、相关记忆源和代码 |
| 多任务并行 | 当前引擎 worktree 规则、变更等级和冲突风险 |

## Token 控制原则

- 先读极薄入口，不读全量项目文档。
- 先用本地检索索引召回摘要、模块、路径和证据。
- 检索命中后，只深读相关代码、测试、项目记忆源或文档。
- S0 微小变更不默认检索项目记忆源，除非用户补充业务背景或涉及风险点。
- S1 小范围任务按模块、文件路径或接口名做小范围检索。
- S2/S3 必须检索相关业务、架构、实现习惯和历史问题记忆。
- 本地索引缺失时，不得假装已掌握上下文；应读取记忆源文本或先重建索引。
- 本地索引只定位上下文，不作为最终事实源。
- 多 Agent 必须有明确触发条件和任务包。

## 上下文回写原则

- 用户补充的业务背景、验收口径、字段含义、状态规则或历史坑，必须评估是否写回项目记忆源。
- 用户补充的前端、后端、客户端或其他端侧风格习惯，必须评估是否写回项目记忆源。
- 协作引擎规则变化只能写回 Agent Context OS 本体或版本化引擎包，不得写入用户项目记忆源。
- 方案仍处于讨论、比较或推敲阶段时，不触发写回；除非用户明确要求记录草案、定版、执行或写回。
- 已确认事实写入项目记忆源；未确认信息写入开放问题或标记为 `draft`。
- Agent 推测只能标记为 `assumption`，不得覆盖 `current`。
- 每次任务报告必须说明本次是否需要记忆源写回和本地索引重建。

## 老项目升级原则

- 首次升级老项目时，目标是原地换芯，不是建立长期兼容层。
- 旧协作引擎规则不迁入项目记忆源；只迁移仍有效的项目业务事实、开发习惯、交互习惯和历史坑。
- 旧文档如果只是历史协作规则，应清退；如果包含业务事实，应先提取为记忆源再清退。
- 升级完成后，后续任务从新版极薄入口和当前协作引擎开始。

## 方案摄取原则

- 方案进入项目时，必须先理解初衷、业务意义、实现逻辑和系统影响。
- 未定版方案只在对话中讨论，不得自动写入项目记忆源。
- 方案全文只作为来源，不作为每次任务默认上下文。
- 方案建议必须标记为 `proposed`，不得直接覆盖 `current`。
- 与当前事实冲突的方案内容必须标记为 `conflict`。

## 变更分级原则

- 修改前先判断 `S0`、`S1`、`S2` 或 `S3`。
- `S0` 只读取入口规则和相关文件，并说明不需要记忆源写回。
- `S1` 读取任务 workflow、相关代码和小范围记忆。
- `S2` 必须评估项目记忆源、业务影响、模块边界和验证影响。
- `S3` 必须评估方案台账、旧引擎清退、决策记录、历史问题和系统影响矩阵。
