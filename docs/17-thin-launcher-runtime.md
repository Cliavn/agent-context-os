# 极薄项目入口与本地记忆索引

## 目标

极薄项目入口用于把协作引擎和用户项目解耦：

- 协作引擎负责 Agent 如何工作、如何拆任务、如何判断门禁、如何控制 Token。
- 用户项目只保留入口配置、可审查的项目记忆源和必要检查脚本。
- 本地检索索引负责快速召回相关记忆，但它是可重建产物，不是团队事实源。

## 三层结构

这套结构也可以简称为 `Memory Source + Local Index`（记忆源 + 本地索引）。

| 层 | 作用 | 推荐位置 |
|---|---|---|
| 协作引擎层 | 任务拆解、上下文路由、S0-S3 门禁、质量规则、多 Agent 策略 | Agent Context OS 本体或版本化引擎包 |
| 项目记忆源 | 业务背景、业务规则、开发习惯、交互习惯、历史坑、证据路径 | 用户项目中的小型 JSONL / Markdown 事实源 |
| 本地检索索引 | 向量、全文索引、关键词索引、过滤元数据 | 用户机器本地缓存目录或 `.agent-context/local-index/` |

## 用户项目默认结构

```text
<项目根目录>/
├─ AGENTS.md
├─ .agent-context/
│  ├─ config.json
│  └─ memory-sources/
│     ├─ README.md
│     └─ *.jsonl
└─ scripts/
   └─ check-agent.ps1
```

用户项目默认不复制完整 `docs/agent/`、workflow、checklist、方案台账或检索库实现。需要这些能力时，由 `config.json` 指向的协作引擎按任务动态加载。

## 项目记忆边界

项目记忆包括：

- 业务背景、业务规则、字段含义、状态流转、验收口径。
- 用户使用习惯、交互习惯、端侧风格、团队开发习惯。
- 模块边界、实现约束、运行限制、验证命令。
- 历史问题、根因、防复发规则和证据路径。

项目记忆不包括：

- 协作引擎的通用 workflow、checklist、任务拆解方式或门禁规则。
- 本地向量库文件、缓存、临时索引或生成产物。
- 可从代码、测试或构建流程重新生成的大文件。
- 账号、密钥、token、cookie 或真实隐私数据。

## 记忆源与索引

项目记忆源是团队共享事实，应进入 Git（版本控制工具），因为它小、可审查、可合并、可追溯。

本地检索索引是查询加速结构，不应进入 Git。它可以由项目记忆源、代码摘要和证据路径重新生成。

```text
项目记忆源 + 代码证据
  ↓
本地索引重建
  ↓
Agent 快速召回相关摘要和路径
  ↓
Agent 阅读真实代码、测试或文档
```

向量数据库中不应只存路径，也不应保存不可审查的大段事实。推荐存储短摘要、关键词、模块、状态、可信度、证据路径和向量。

## 老项目升级

老项目升级应采用原地换芯：

1. 识别旧协作引擎文件和项目记忆内容。
2. 将仍有效的业务事实、开发习惯、交互习惯和历史坑提取为项目记忆源。
3. 清退旧协作引擎文件，不长期保留两套协作结构。
4. 写入极薄项目入口和 `.agent-context/config.json`。
5. 重建本地检索索引。
6. 运行项目检查和当前引擎要求的强检查。

升级只替换协作引擎逻辑，不迁移本地向量库文件。索引结构变化时直接重建。

## 门禁

- 未完成项目记忆源校验时，Agent 不得执行依赖业务上下文的 S2/S3 任务。
- 本地索引缺失时，Agent 可以先检索记忆源文本；如果任务需要高效召回，应先重建索引。
- 检索命中只是线索，高风险内容必须回到代码、测试、文档或用户确认验证。
- 本地索引目录必须被 `.gitignore` 排除。
- 同一事实只能有一个当前记忆源；过期内容标记为 `stale` 或 `deprecated`。
