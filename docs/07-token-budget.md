# Token 成本控制

## 目标

Token 成本控制用于在提高代码质量的同时，避免上下文消耗失控。

## 默认策略

- 默认单 Agent。
- 默认读取入口、路由、任务 workflow、模块卡片和必要业务摘要。
- 默认不读取历史长文档。
- 默认不启用多 Agent。
- 默认不全仓扫描，除非任务需要。

## 预算分级

| 等级 | 适用场景 | 上下文策略 |
|---|---|---|
| S | 小修小补 | 入口 + 相关文件 |
| M | 普通功能或 Bug | workflow + 模块卡片 + 相关代码 |
| L | 跨模块任务 | 增加架构、业务摘要、测试 |
| XL | 复杂排查或重构 | 深读历史问题，必要时启用多 Agent |

## 升级条件

只有出现以下情况才升级上下文：

- 文档和代码冲突。
- 复现路径不清。
- 涉及多个模块或多个业务对象。
- 涉及历史复发问题。
- 涉及数据、权限、安全或不可逆操作。

## 成本报告

复杂任务最终报告应说明：

- 读取了哪些文档。
- 为什么需要深读。
- 是否启用了多 Agent。
- 是否存在可优化的上下文入口。
