# Token 成本控制

## 目标

Token 成本控制用于在提高代码质量的同时，避免上下文消耗失控。

## 默认策略

- 默认单 Agent。
- 默认读取入口、路由、任务 workflow、模块卡片和必要业务摘要。
- 默认不读取历史长文档。
- 默认不启用多 Agent。
- 默认不全仓扫描，除非任务需要。
- 老项目首次接入默认只建立旧文档索引，不读取全文。
- 完整方案默认先生成结构化摄取记录，不作为后续任务默认全文上下文。
- 微小变更默认走 S0 轻量模式，不读取无关项目记忆、项目风格画像和长文档。

## 预算分级

| 等级 | 适用场景 | 上下文策略 |
|---|---|---|
| S0 | 微小样式、文案或局部无业务变更 | AGENTS + 相关文件 + 最小验证 |
| S | 小修小补 | 入口 + 相关文件 |
| M | 普通功能或 Bug | workflow + 模块卡片 + 相关代码 |
| L | 跨模块任务 | 增加架构、业务摘要、测试 |
| XL | 复杂排查或重构 | 深读历史问题，必要时启用多 Agent |
| ADOPT | 老项目首次接入 | 入口 + 接入 workflow + 最小模板 + 旧文档索引 |
| INTAKE | 完整方案摄取 | 方案全文一次性理解 + 结构化索引 + 系统影响矩阵 |

## 升级条件

只有出现以下情况才升级上下文：

- 文档和代码冲突。
- 复现路径不清。
- 涉及多个模块或多个业务对象。
- 涉及历史复发问题。
- 涉及数据、权限、安全或不可逆操作。
- 渐进式接入后，具体任务首次触达某个旧文档对应模块。
- 方案摄取后，具体任务首次触达方案影响的模块、业务对象或接口。
- S0 修改触发业务规则、字段、状态、接口、权限或模块边界变化。
- S0 修改触发项目风格画像、端侧习惯或工程约定变化。

## 成本报告

复杂任务最终报告应说明：

- 读取了哪些文档。
- 为什么需要深读。
- 是否启用了多 Agent。
- 是否存在可优化的上下文入口。
- 是否读取旧文档全文；如果读取，说明触发原因。
- 是否读取方案全文；如果读取，说明为什么不能只读摄取索引。
- 本次变更等级；如果是 S0，说明为什么不需要文档同步。
