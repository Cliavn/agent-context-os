# 执行门禁与变更分级

## 目标

执行门禁与变更分级用于保证 Agent Context Engine 既能约束高风险变更，又不会让微小改动承担过重成本。

核心目标：

- 小事轻：微小样式、文案和局部调整走轻量模式。
- 大事稳：业务、模块、方案、架构和高风险变更走完整流程。
- 不偏移：代码修改后必须评估项目记忆、业务文档和模块文档是否需要同步。
- 可检查：不依赖 Agent 自觉，关键规则应能被脚本、提交门禁或 CI 检查。

## 变更等级

| 等级 | 名称 | 示例 | 默认流程 |
|---|---|---|---|
| `S0` | 微小变更 | 按钮颜色、间距、错别字、纯样式 | 轻量模式 |
| `S1` | 局部变更 | 单组件交互、小范围 Bug、局部逻辑 | 标准局部流程 |
| `S2` | 业务或模块变更 | 字段、状态、接口、权限、模块边界 | 完整记忆和文档同步 |
| `S3` | 方案或架构变更 | 已确认进入摄取或执行的完整方案、跨模块重构、老项目接入、高风险流程 | 完整 Engine 流程 |

## S0 轻量模式

S0 适用于不影响业务规则、数据、接口、权限、状态和模块边界的微小变更。

S0 必须做：

- 阅读 `AGENTS.md`。
- 查看相关代码或样式文件。
- 执行最小验证或说明无法验证原因。
- 最终说明不涉及记忆回写、方案摄取、旧文档迁移和业务文档同步。

S0 不需要默认读取：

- `docs/agent/memory.md`
- `docs/agent/intake.md`
- `docs/agent/legacy-docs.md`
- 业务语义长文档
- 历史问题库

## 升级条件

S0 或 S1 任务出现以下任一情况，必须升级：

- 涉及业务规则、验收口径、字段语义或状态变化。
- 涉及项目风格画像、端侧交互习惯、命名偏好或工程约定变化。
- 涉及 API、数据库、权限、配置或自动化流程。
- 涉及模块职责、依赖方向或架构边界。
- 用户补充了可复用业务背景。
- 任务来源于已确认摄取或执行的完整方案。
- 发现文档、代码或测试存在冲突。
- 修复重复 Bug 或历史坑。
- 存在不可逆操作、数据迁移或高风险发布影响。

## 任务状态门禁

S1 及以上任务建议维护运行态任务文件：

```text
docs/agent/runtime/current-task.md
```

S2 和 S3 必须维护运行态任务文件。

运行态任务文件用于记录：

- `change_level`
- `task_type`
- `docs_checked`
- `memory_writeback`
- `style_profile`
- `plan_intake`
- `legacy_docs`
- `verification`
- `git_commit`
- `open_questions`

## drift 检查

文档偏移检查应关注：

- 代码变更是否有任务状态或任务报告。
- S2/S3 是否评估记忆回写。
- 方案相关变更是否引用 `intake.md`。
- 老项目接入变更是否更新 `adoption.md` 或 `legacy-docs.md`。
- 业务规则、字段、状态、接口或权限变化是否更新对应文档。
- 项目风格画像、端侧习惯或命名约定变化是否更新对应文档。

## 门禁策略

| 等级 | 读取范围 | 文档同步 | 本地提交 | 报告要求 | 脚本检查 |
|---|---|---|---|---|---|
| `S0` | 入口 + 相关文件 | 说明不需要 | 修改文件后默认提交 | 极简说明 | 允许轻量通过 |
| `S1` | 入口 + 任务 workflow + 相关模块 | 评估是否需要 | 默认提交 | 标准摘要 | 检查任务状态建议 |
| `S2` | 增加 memory、业务、模块文档 | 必须明确写回或说明不需要 | 默认提交并记录状态 | 完整任务报告 | 检查任务状态必需 |
| `S3` | 增加 intake、legacy、决策、历史问题 | 必须完整记录 | 默认提交并记录状态 | 完整报告和影响矩阵 | 强检查 |

## 禁止事项

- 不得用完整流程压迫 S0 微小变更。
- 不得把 S2/S3 伪装成 S0 来绕过文档同步。
- 不得只依赖 Agent 口头承诺遵守规则。
- 不得让代码变更在没有文档同步评估的情况下进入交付。
- 不得在方案讨论阶段修改协作文档或把未定版内容写成当前事实。
- 不得把用户已有未提交改动混入本次本地提交。
- 不得在未获得用户明确授权时推送到远端仓库。
- 不得省略最终回复结尾的本地提交状态和“是否推送”。
