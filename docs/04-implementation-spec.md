# 开发实现说明

## 目标

开发实现说明用于在写代码前把需求、边界、影响范围、历史风险和验证方式说清楚，从源头降低 Bug 产出。

## 适用场景

- 新增功能
- Bug 修复
- 重构
- 字段规则变更
- 接口变更
- 数据库变更
- 自动化流程变更
- UI 交互变更
- 完整方案拆解后的实施任务

## 标准结构

```md
# 开发实现说明：<任务名称>

## 1. 修改目标

## 2. 非目标

## 3. 变更等级

## 4. 业务规则引用

## 5. 项目风格画像引用

## 6. 当前实现理解

## 7. 核心方案

## 8. 影响范围

## 9. 数据流与状态流

## 10. 历史问题检查

## 11. 记忆回写计划

## 12. 方案摄取引用

## 13. 验证矩阵

## 14. 风险与回滚
```

## 把关规则

- 没有明确目标，不进入实现。
- 没有非目标，容易扩大范围。
- 没有变更等级，无法判断轻量模式还是完整门禁。
- 涉及业务规则时，必须引用业务语义文档。
- 涉及视觉、交互、命名、分层或端侧习惯时，必须引用项目风格画像。
- 涉及状态或数据写入时，必须写数据流与状态流。
- 涉及历史问题时，必须检查问题库。
- 涉及用户补充背景、业务规则变化或开放问题时，必须写记忆回写计划。
- 来源于完整方案的任务，必须引用方案摄取记录和系统影响矩阵。
- 没有验证矩阵，不视为可交付。
