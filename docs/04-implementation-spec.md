# 开发实现说明

## 目标

开发实现说明用于在写代码前把需求、边界、影响范围、历史风险和验证方式说清楚，从源头降低 Bug 产出。

## 适用场景

- 新增功能
- Bug 修复
- 重构
- 字段规则变更
- 接口变更
- 数据库变更
- 自动化流程变更
- UI 交互变更

## 标准结构

```md
# 开发实现说明：<任务名称>

## 1. 修改目标

## 2. 非目标

## 3. 业务规则引用

## 4. 当前实现理解

## 5. 核心方案

## 6. 影响范围

## 7. 数据流与状态流

## 8. 历史问题检查

## 9. 验证矩阵

## 10. 风险与回滚
```

## 把关规则

- 没有明确目标，不进入实现。
- 没有非目标，容易扩大范围。
- 涉及业务规则时，必须引用业务语义文档。
- 涉及状态或数据写入时，必须写数据流与状态流。
- 涉及历史问题时，必须检查问题库。
- 没有验证矩阵，不视为可交付。
