# <项目名> Agent 协作入口

本文件是 `<项目名>` 的极薄 Agent 入口。项目不内置完整协作引擎；协作引擎的任务拆解、上下文路由、执行门禁和质量规则由 `Agent Context OS` 统一管理。

## 启动顺序

1. 读取本文件。
2. 读取 `.agent-context/config.json`。
3. 按 `engine.source` 加载当前协作引擎。
4. 按 `memory.source_paths` 检查项目记忆源。
5. 按需使用本地检索索引；本地索引缺失或过期时先重建。

## 硬规则

- 用户项目只保存极薄入口、项目配置和可审查的记忆源，不保存完整协作引擎副本。
- 协作引擎规则、S0-S3 变更等级、任务 workflow、执行门禁和质量门禁以 `engine.source` 指向的当前引擎为准。
- 项目记忆源是团队共享事实；本地向量库或全文索引只是可重建的检索索引，不得作为唯一事实源。
- 本地检索索引不得提交到 Git（版本控制工具）。
- 涉及业务、交互、架构、实现习惯或历史坑时，Agent 必须先检索项目记忆源，再按命中结果读取真实代码、测试或文档。
- 检索结果只负责定位上下文；与代码、测试或用户确认冲突时，以代码、测试和用户确认为准。
- 用户补充可复用事实时，应写入项目记忆源或由当前引擎指定的团队记忆服务，不得写入旧版协作文档。
- 老项目升级时，应先从旧协作引擎中提取仍有效的项目记忆，再清退旧协作引擎文件；不得长期保留两套协作结构。
- 未完成记忆源校验或本地索引重建前，Agent 不得假装已经掌握完整业务上下文。
- 修改后运行 `scripts/check-agent.ps1`，并按当前协作引擎要求执行必要验证。
- 完成修改并通过必要验证后，默认创建本地 Git 提交；不得在未获得明确授权时推送远端。
- 不得把账号、密钥、token、cookie、真实隐私数据写入配置、记忆源、文档、代码或测试。

## 本地索引

本地索引默认放在用户机器的缓存目录或 `.agent-context/local-index/`。这些文件可以删除并重建，必须由 `.gitignore` 排除。

## 最终报告

最终回复至少说明：

- 使用的协作引擎版本或来源
- 是否检索项目记忆源
- 本地索引状态
- 变更等级
- 修改范围
- 验证结果
- 本地 Git 提交
- 是否推送
