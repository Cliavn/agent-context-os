# 方案摄取索引

本文件用于记录 `<项目名>` 中由用户输入的业务方案、产品方案、技术方案或执行计划。

方案进入项目后，不应只保存全文。Agent 必须先理解方案意图和业务意义，再拆解系统影响，并把有效内容写入对应上下文。

如果用户只是讨论、比较或推敲方案，且未明确要求定版、执行、记录草案或写回，Agent 不得修改本文件或其他协作文档；只能在对话中整理候选方案、风险和待确认问题。

## 方案状态

| 状态 | 含义 | 使用规则 |
|---|---|---|
| `current` | 已确认并作为当前事实 | 可作为实现依据 |
| `proposed` | 方案建议，尚未落地或确认 | 不得覆盖当前事实 |
| `assumption` | Agent 推测出的内容 | 必须等待用户确认 |
| `open` | 需要用户或负责人确认 | 进入开放问题 |
| `conflict` | 与现有事实、代码或测试冲突 | 必须报告并处理 |
| `deprecated` | 已被新方案替代 | 只用于回溯 |

## 规则

- 只有明确要求定版、执行、记录草案或写回的方案，才能进入本文件。
- 方案全文不得作为默认上下文。
- 方案建议不得直接覆盖当前事实。
- 方案中的 Agent 推测必须标记为 `assumption`。
- 与现有事实冲突的内容必须标记为 `conflict`。
- 每个方案必须保留来源和摄取报告。
