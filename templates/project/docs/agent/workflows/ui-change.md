# Workflow：UI 改造

## 目标

UI 改造必须服务真实用户动作，保持视觉、交互和业务状态一致。

## 步骤

1. 阅读入口、前端模块文档、质量规则和本 workflow。
2. 明确改动对象、用户动作、状态组合和验收标准。
3. 查找现有组件、样式、交互模式和测试。
4. 明确非目标，避免顺手重做页面。
5. 实现最小必要修改。
6. 验证默认态、加载态、错误态、空态和交互态。
7. 运行相关前端验证。
8. 按任务报告模板汇报。

## 禁止事项

- 不允许只验证默认静态页面。
- 不允许让文案、按钮或表格在常见尺寸下溢出。
- 不允许把业务状态只保存在前端临时状态里。
