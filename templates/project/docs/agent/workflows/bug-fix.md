# Workflow：修复 Bug

## 目标

修复 Bug 时，必须先理解现象、复现路径和根因，再做最小必要修改。

## 步骤

1. 阅读入口、质量规则、相关模块文档和本 workflow。
2. 明确期望行为、实际行为和影响范围。
3. 查找相关代码、测试、日志或错误信息。
4. 尽量复现问题；无法复现时说明限制。
5. 定位根因，区分真实修复和表面规避。
6. 检查历史问题库，确认是否为复发问题。
7. 做最小必要修改。
8. 补充或更新回归测试。
9. 运行相关验证。
10. 若暴露规则缺口，更新或建议更新文档、测试或 checklist。
11. 按任务报告模板汇报。

## 禁止事项

- 不允许在未定位根因时大面积重写。
- 不允许删除或弱化测试来让结果通过。
- 不允许隐藏错误或吞掉异常。
- 不允许把修复范围扩大到无关重构。
