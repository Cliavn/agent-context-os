# Workflow：执行门禁

## 目标

在修改前先判断变更等级，并按等级决定读取范围、文档同步和验证强度。

## 步骤

1. 阅读 `AGENTS.md` 和本 workflow。
2. 判断变更等级：`S0`、`S1`、`S2` 或 `S3`。
3. 如果是 `S0`，走轻量模式，只读取相关文件并做最小验证；涉及视觉、交互、命名或端侧习惯时可补读 `style-profile.md`。
4. 如果是 `S1`，读取任务 workflow 和相关模块上下文。
5. 如果是 `S2`，补充读取 `memory.md`、业务文档和模块文档。
6. 如果是 `S3`，补充读取 `intake.md`、`legacy-docs.md`、决策记录和历史问题。
7. 判断是否需要维护 `docs/agent/runtime/current-task.md`。
8. 如果任务来自方案讨论（`discussion_only`），先确认是否已定版、执行、记录草案或写回；未确认时不得修改协作文档。
9. 完成修改后，评估记忆回写、风格画像回写、方案摄取、旧文档索引和文档同步。
10. 运行对应验证。
11. 输出匹配变更等级的任务报告。

## 禁止事项

- 不允许把高风险任务降级为 `S0`。
- 不允许让 `S0` 任务承担完整流程成本。
- 不允许在方案讨论阶段修改协作文档。
- 不允许跳过文档同步评估直接交付代码。
- 不允许只依赖口头说明，不留下可检查痕迹。
