# Workflow：重构

## 目标

重构必须保持外部行为不变，并降低真实复杂度。

## 步骤

1. 阅读入口、项目风格画像、项目记忆、架构、相关模块文档和质量规则。
2. 明确重构目标和非目标。
3. 明确受影响的公共接口、数据结构和测试。
4. 保留行为保持证据。
5. 分小步修改，避免大面积同时改变行为和结构。
6. 运行相关测试和构建检查。
7. 更新受影响的模块文档。
8. 若重构暴露新的模块边界、命名规则、端侧工程习惯或历史约束，写回项目记忆、项目风格画像或架构文档。
9. 按任务报告模板汇报行为保持证据。

## 禁止事项

- 不允许借重构改变业务语义。
- 不允许同时引入大量无关风格调整。
- 不允许删除验证覆盖。
- 不允许把新发现的架构事实只留在对话里。
