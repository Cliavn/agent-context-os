# Workflow：新增功能

## 目标

新增功能时，必须先确认需求、业务规则、模块边界和验收标准，再实现代码。

## 步骤

1. 阅读入口、变更分级、项目画像、项目风格画像、项目记忆、架构、技术栈和质量规则。
2. 判断变更等级，确认是否需要完整流程。
3. 找到相关模块文档和业务语义文档。
4. 如果需求来自完整方案，先阅读方案摄取记录和系统影响矩阵。
5. 查找现有实现、相似功能、测试、命名风格和端侧约定。
6. 明确输入、输出、错误路径、配置需求和验收标准。
7. 输出或更新开发实现说明。
8. 选择最小实现路径。
9. 实现功能，并补充必要测试。
10. 运行相关验证。
11. 若新增公共接口、字段、状态、配置或模块，更新对应文档。
12. 若用户补充业务背景、风格偏好或产生新项目事实，写回项目记忆、项目风格画像或记录开放问题。
13. 按任务报告模板汇报。

## 禁止事项

- 不允许跳过现有实现搜索直接写新代码。
- 不允许绕过模块边界。
- 不允许引入未记录的新依赖。
- 不允许把用户确认的业务事实只留在对话里。
- 不允许跳过方案摄取记录直接按方案全文实现。
- 不允许把临时实验代码留在主路径。
