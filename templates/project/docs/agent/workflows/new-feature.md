# Workflow：新增功能

## 目标

新增功能时，必须先确认需求、业务规则、模块边界和验收标准，再实现代码。

## 步骤

1. 阅读入口、项目画像、架构、技术栈和质量规则。
2. 找到相关模块文档和业务语义文档。
3. 查找现有实现、相似功能、测试和命名风格。
4. 明确输入、输出、错误路径、配置需求和验收标准。
5. 输出或更新开发实现说明。
6. 选择最小实现路径。
7. 实现功能，并补充必要测试。
8. 运行相关验证。
9. 若新增公共接口、字段、状态、配置或模块，更新对应文档。
10. 按任务报告模板汇报。

## 禁止事项

- 不允许跳过现有实现搜索直接写新代码。
- 不允许绕过模块边界。
- 不允许引入未记录的新依赖。
- 不允许把临时实验代码留在主路径。
