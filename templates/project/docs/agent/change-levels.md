# 变更分级

本文件定义 `<项目名>` 的变更等级和对应 Agent Context Engine 流程强度。

## 等级定义

| 等级 | 名称 | 示例 | 默认流程 |
|---|---|---|---|
| `S0` | 微小变更 | 按钮颜色、间距、错别字、纯样式 | 轻量模式 |
| `S1` | 局部变更 | 单组件交互、小范围 Bug、局部逻辑 | 标准局部流程 |
| `S2` | 业务或模块变更 | 字段、状态、接口、权限、模块边界 | 完整记忆和文档同步 |
| `S3` | 方案或架构变更 | 完整方案、跨模块重构、老项目接入、高风险流程 | 完整 Engine 流程 |

## S0 轻量模式

S0 只需要：

- 阅读 `AGENTS.md`。
- 查看相关代码或样式文件。
- 执行最小验证或说明原因。
- 最终说明不涉及记忆回写、方案摄取、旧文档迁移和业务文档同步。

S0 默认不读取：

- `docs/agent/memory.md`
- `docs/agent/intake.md`
- `docs/agent/legacy-docs.md`
- 业务语义长文档
- 历史问题库

## 升级条件

出现以下情况必须升级：

- 涉及业务规则、字段语义、状态或验收口径。
- 涉及项目风格画像、端侧交互习惯、命名偏好或工程约定变化。
- 涉及 API、数据库、权限、配置或自动化流程。
- 涉及模块职责、依赖方向或架构边界。
- 用户补充了可复用业务背景。
- 任务来源于完整方案。
- 发现文档、代码或测试冲突。
- 修复重复 Bug 或历史坑。

## 报告要求

| 等级 | 报告要求 |
|---|---|
| `S0` | 极简说明：改了什么、验证什么、为什么不需要文档同步 |
| `S1` | 标准摘要：已读上下文、修改范围、验证和风险 |
| `S2` | 完整报告：记忆回写、业务文档、模块文档和验证 |
| `S3` | 完整报告 + 系统影响矩阵 + 开放问题 |
