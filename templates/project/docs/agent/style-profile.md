# 项目风格画像

本文件记录 `<项目名>` 的项目级风格、端侧习惯和工程约定。它不是通用规范，也不用于约束其他项目。

## 使用规则

- 只有已确认事实才能标记为 `current`。
- 信息不完整时标记为 `draft` 或写入开放问题。
- Agent 推测只能标记为 `assumption`，不得作为实现依据。
- 不得把某个页面、接口或模块的一次性做法提升为全项目风格。
- 修改视觉、交互、文案、命名、分层或端侧习惯时，应先检查本文件。

## 风格状态

| 状态 | 含义 | 使用规则 |
|---|---|---|
| `current` | 已确认的当前项目风格 | 可作为实现依据 |
| `draft` | 方向基本明确，但细节未完整确认 | 使用前需要确认关键细节 |
| `assumption` | Agent 为推进任务做出的假设 | 不得作为最终事实沉淀 |
| `deprecated` | 已废弃或被替代 | 只用于回溯 |

## 回写规则

- 用户补充可复用风格、交互、文案、命名或工程习惯时，必须评估是否写回本文件。
- 新功能沉淀出可复用端侧模式时，写回对应端侧表格。
- 重构改变命名、分层或组织方式时，写回后端或相关端侧表格。
- UI 改造只影响单个局部且不会复用时，不必提升为项目风格。
- 旧文档中的风格约定必须经过代码、测试或用户确认后才能迁移为 `current`。
