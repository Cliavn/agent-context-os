# 项目记忆

本文件用于沉淀 `<项目名>` 在协作过程中逐步形成的项目事实、业务事实、用户补充背景和开放问题。

Agent 在任务结束前必须检查本次对话或修改是否产生了需要写回的记忆。

如果项目已启用 `docs/agent/memory-store/`，Agent 应优先通过检索式记忆召回相关上下文，再按命中结果读取本文件或对应事实源。本文件仍用于维护人类可审查的项目记忆摘要和开放问题。

## 记忆状态

| 状态 | 含义 | 使用规则 |
|---|---|---|
| `current` | 已确认的当前事实 | 可作为实现依据 |
| `draft` | 方向基本明确，但细节未完整确认 | 使用前需要确认关键细节 |
| `assumption` | Agent 为推进任务做出的假设 | 不得作为最终事实沉淀 |
| `deprecated` | 已废弃或被替代 | 只用于回溯 |
| `stale` | 疑似过期或与当前实现冲突 | 使用前必须重新验证 |

## 检索式记忆索引

结构化检索记忆维护在 `docs/agent/memory-store/`。

| 位置 | 用途 | 维护规则 |
|---|---|---|
| `memory-store/memories.jsonl` | 保存可导出的记忆记录 | 新增、更新、废弃和校验记忆时同步维护 |
| `memory-store/memory-schema.json` | 定义单条记忆字段 | 字段规则变化时更新 |
| `memory-store/retrieval-config.json` | 定义检索、过滤、排序和过期策略 | 检索策略变化时更新 |

检索记忆不是唯一事实来源。当前事实仍应同步维护在项目文档、业务文档、模块文档、决策记录或测试中。

## 记忆回写规则

- 用户补充业务背景、业务规则、验收口径或历史坑时，必须评估是否写回。
- 用户补充项目风格、端侧习惯、命名偏好或工程约定时，必须评估是否写回 `style-profile.md`。
- 用户纠正 Agent 理解时，必须更新对应事实或记录开放问题。
- 用户只是在讨论、比较或推敲方案时，不得把未定版方向写回当前事实源。
- 涉及字段、状态、接口、权限、流程、模块边界或验证命令变化时，必须更新相关文档。
- 启用检索式记忆后，涉及可复用事实、历史坑、交互约束或实现约束时，必须评估是否同步更新 `memory-store/memories.jsonl`。
- Agent 只能把已确认事实标记为 `current`。
- 信息不完整时，记录为 `draft` 或开放问题，并主动追问。
- 临时假设只能标记为 `assumption`，不得覆盖当前事实。
- 不得记录账号、密钥、token、cookie 或真实隐私数据。
- 检索记忆与代码、测试或当前文档冲突时，应标记为 `stale` 或 `deprecated`，不得继续作为当前事实使用。

## 任务结束前检查

- 本次是否出现新的可复用项目事实或业务事实。
- 是否已写回 `docs/agent/`、`style-profile.md`、业务文档、模块文档、历史问题库或 checklist。
- 没有写回时，原因是否明确。
- 是否存在需要用户确认的开放问题。
- 如果启用了 `memory-store/`，本次是否新增、更新、废弃或校验了检索记忆。
