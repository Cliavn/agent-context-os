# Checklist：修复 Bug

- [ ] 已阅读入口、变更分级、项目记忆、相关模块文档、质量规则和 bug fix workflow。
- [ ] 已判断变更等级，并确认是否需要升级。
- [ ] 已明确期望行为和实际行为。
- [ ] 已尝试复现问题，或说明无法复现原因。
- [ ] 已定位根因。
- [ ] 已检查历史问题库。
- [ ] 修改范围保持最小。
- [ ] 已补充或更新回归测试，或说明无法补充原因。
- [ ] 已运行相关验证。
- [ ] 未删除或弱化测试来掩盖问题。
- [ ] 已评估复现背景、历史坑或规则缺口是否需要记忆回写。
- [ ] 如涉及视觉、交互、命名或端侧习惯，已评估是否需要写回项目风格画像。
- [ ] 如存在未确认信息，已记录开放问题或主动追问。
- [ ] 最终报告包含根因、修改、验证和风险。
