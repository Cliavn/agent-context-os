# Checklist：方案理解与摄取

- [ ] 已确认方案来源、类型和适用范围。
- [ ] 已确认当前不是仅讨论阶段（`discussion_only`），或仅讨论阶段已停止文档写回。
- [ ] 如进入摄取，已获得用户对定版、执行、记录草案或写回的明确确认。
- [ ] 如用户要求形成开发顺序或任务清单，已创建或更新 `docs/agent/plans/<plan-id>.md`。
- [ ] 已说明不会只保存方案全文作为默认上下文。
- [ ] 已提取方案初衷、业务痛点、成功标准和非目标。
- [ ] 已提取用户角色、业务对象、流程、字段、状态、异常和验收口径。
- [ ] 已分析模块、接口、数据、权限、配置、运行态、验证和项目风格影响。
- [ ] 已形成开发顺序，并为每个任务记录任务 ID、来源摘要、依赖、验收口径和状态。
- [ ] 已生成系统影响矩阵。
- [ ] 已检查与当前记忆、旧文档、代码事实或测试结果是否冲突。
- [ ] 已区分 `current`、`proposed`、`assumption`、`open` 和 `conflict`。
- [ ] 已按确认范围将有效事实写回对应上下文；记录草案未覆盖 `current`。
- [ ] 已确认 `draft` 台账没有被当作开发依据。
- [ ] 已记录开放问题。
- [ ] 已输出方案摄取报告。
