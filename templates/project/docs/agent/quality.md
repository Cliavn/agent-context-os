# 质量门禁

## 代码质量

- 修改范围必须与需求匹配。
- 优先复用现有结构、命名、接口和工具链。
- 不引入无必要的新抽象。
- 用户补充的可复用项目事实必须评估是否写回项目记忆。
- 用户补充的可复用风格事实必须评估是否写回项目风格画像。
- 老项目首次接入不得全量分析旧文档。
- 完整方案不得只保存为单一长文本，必须先做结构化摄取。
- 方案仍处于讨论阶段时，不得修改协作文档；必须等用户明确要求定版、执行、记录草案或写回。
- 完整方案落地不得只依赖对话记忆，必须使用 `confirmed` 或 `active` 方案落实台账。
- 微小变更应走 S0 轻量模式，不得强行消耗完整流程成本。
- 错误处理必须可观察、可恢复或可明确失败。
- 状态、缓存、并发、文件系统、网络和数据库逻辑必须有明确生命周期。

## 测试标准

- 新增业务逻辑优先补单元测试。
- 修复 Bug 优先补回归测试。
- 跨模块流程补集成测试或手工验收。
- 重构必须证明外部行为不变。
- 无法测试时必须说明原因、手工验证方式和剩余风险。

## 完成定义

一次任务完成必须满足：

- 已阅读相关上下文。
- 已判断变更等级。
- 已遵守模块边界。
- 已执行或说明无法执行的验证。
- 已评估文档同步需求。
- 已评估记忆回写需求。
- 已评估渐进式接入状态或旧文档索引是否需要更新。
- 已评估方案摄取状态或系统影响矩阵是否需要更新。
- 已在最终报告中说明风险。

## 变更分级

- `S0`：微小变更，只需入口规则、相关文件和最小验证。
- `S1`：局部变更，需要任务 workflow 和相关模块上下文。
- `S2`：业务或模块变更，需要记忆回写和文档同步评估。
- `S3`：方案或架构变更，需要完整 Engine 流程和系统影响矩阵。

## 记忆回写

- 新业务背景、验收口径、字段含义、状态规则或历史坑应写回 `docs/agent/memory.md`、业务文档或模块文档。
- 新视觉、交互、命名、分层或端侧习惯应写回 `docs/agent/style-profile.md`、业务文档或模块文档。
- 未确认信息应记录为开放问题，不得写成当前事实。
- Agent 假设必须标记为 `assumption`，并等待用户确认。

## 渐进式接入

- 首次接入只安装最小协作规则，不全量整理旧文档。
- 旧文档先写入 `docs/agent/legacy-docs.md`，状态默认为 `indexed`，可信度默认为 `unknown`。
- 只有任务需要时，才读取旧文档全文并迁移有效事实。

## 方案摄取

- 完整方案只有在用户明确要求定版、执行、记录草案或写回后，才写入 `docs/agent/intake.md` 的结构化索引。
- 讨论阶段只在对话中输出候选方案、风险和待确认问题，不修改 `docs/agent/`、业务文档、模块文档或决策记录。
- 方案必须先拆出意图、业务意义、实现逻辑、系统影响和融合更新。
- 方案建议默认标记为 `proposed`，不得直接覆盖 `current`。

## 方案落实台账

- 用户要求形成开发顺序、任务清单、记录草稿或准备落地时，创建或更新 `docs/agent/plans/<plan-id>.md`。
- 台账初始状态为 `draft`，用户确认后才能改为 `confirmed`。
- 用户要求按完整方案开发时，必须按 `confirmed` 或 `active` 台账逐项执行。
- 最终报告必须列出任务总数、完成、阻塞、延期和未开始数量。
- 仍存在无说明的 `pending` 任务时，不得宣称完整方案已交付。

## 执行门禁

- S0 必须说明不涉及业务规则、记忆回写、方案摄取和旧文档迁移。
- S2/S3 必须维护或更新 `docs/agent/runtime/current-task.md`。
- 启用 `docs/agent/memory-store/` 时，应运行 `scripts/check-project-memory-store.ps1`。
- 代码、脚本或配置修改后应运行 `scripts/check-agent-drift.ps1`。
- 代码、脚本、模板或协作文档修改后应运行强检查矩阵；优先使用 `scripts/check-agent-strong.ps1`。
- 无法运行项目检查脚本时，必须说明原因、替代验证方式和剩余风险。

## 强检查矩阵

任务收尾时优先运行：

```powershell
scripts/check-agent-strong.ps1
```

强检查至少覆盖：

- `git diff --check`：检查未暂存差异中的空白错误。
- `git diff --cached --check`：检查已暂存差异中的空白错误。
- `scripts/check-project-memory-store.ps1`：检查检索式项目记忆结构。
- `scripts/check-agent-worktrees.ps1`：检查 Git worktree（工作树）残留和嵌套风险。
- `scripts/check-agent-drift.ps1`：检查代码类变更是否留下任务状态和文档同步痕迹。

如果某个检查不适用于当前项目，最终报告必须说明跳过原因；如果检查失败，不得宣称交付完成，除非用户明确接受失败状态和剩余风险。
