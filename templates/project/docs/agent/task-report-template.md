# 任务报告模板

Agent 完成任何代码修改、文档修改或评审后，最终回复应包含以下信息。可以根据任务大小压缩，但不得省略验证和风险。

```md
已完成：<一句话说明>

已读文档：
- AGENTS.md
- docs/agent/00-index.md
- <其他相关文档>

修改范围：
- <路径或模块>：<说明>

变更等级：
- 等级：<S0 / S1 / S2 / S3>
- 原因：<为什么是这个等级>
- 是否触发升级：<否 / 升级原因>

模块边界影响：
- <未触碰 / 触碰了哪些边界以及原因>

项目风格画像：
- <未涉及 / 已遵守 / 已更新 docs/agent/style-profile.md / 建议更新>

验证结果：
- 已运行：<命令或检查方式>
- 未运行：<原因>

本地 Git 提交：
- 状态：<已提交 / 跳过 / 不适用>
- 提交：<commit hash / 跳过原因>
- 是否推送（pushed）：<是 / 否 / 未授权 / 不适用>

文档同步：
- <是否需要更新协作文档、业务文档或问题库>

记忆回写：
- 本次新增事实：<无 / 事实摘要>
- 写回位置：<无 / docs/agent/memory.md / 业务文档 / 模块文档 / 问题库>
- 开放问题：<无 / 需要用户确认的问题>

检索记忆：
- 召回状态：<未启用 / 未检索 / 已检索>
- 命中摘要：<无 / 相关记忆摘要>
- 更新状态：<无 / 新增 / 更新 / 废弃 / 校验>
- 写回位置：<无 / docs/agent/memory-store/memories.jsonl / 外部数据库>
- 冲突或过期：<无 / 说明>

渐进式接入状态：
- 接入状态变化：<无 / progressive / mature>
- 旧文档读取：<无 / 路径与原因>
- 旧文档索引更新：<无 / 更新说明>

方案摄取状态：
- 关联方案：<无 / plan-id>
- 方案状态：<无 / current / proposed / assumption / open / conflict>
- 系统影响矩阵：<无 / 摘要或路径>
- 开放问题：<无 / 待确认问题>

执行门禁：
- current-task：<未使用 / 已更新 / 已归档>
- drift 检查：<未运行 / 通过 / 失败 / 不适用>

风险与后续：
- <剩余风险>
- <建议后续动作>

本地提交状态：<已提交 / 跳过 / 不适用>；是否推送（pushed）：<是 / 否 / 未授权 / 不适用>
```
