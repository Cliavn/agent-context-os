# 检索式项目记忆存储

本目录用于保存 `<项目名>` 的可检索项目记忆。它是 `docs/agent/memory.md` 的召回层，不是唯一事实来源。

事实优先级：

```text
代码和测试 > 当前文档 > 决策记录 > 检索记忆 > 对话摘要
```

## 文件说明

| 文件 | 说明 |
|---|---|
| `memory-schema.json` | 单条记忆的 JSON Schema |
| `memories.jsonl` | 可导出的逐行 JSON 记忆记录 |
| `retrieval-config.json` | 检索、过滤、排序和过期策略 |

## 使用规则

- 任务开始时，按任务类型、模块、文件路径、接口名或业务对象检索相关记忆。
- `S0` 微小变更默认不检索，除非涉及业务规则、历史坑或用户补充背景。
- `S2` / `S3` 变更必须检索相关业务、架构、实现、历史问题和决策记忆。
- 检索命中后，不得直接照搬；高风险内容必须回到代码、测试或当前文档验证。
- 当前事实必须同步维护在 `docs/agent/*.md`、业务文档、模块文档或决策记录中。
- 发现过期或冲突记忆时，保留原记录并把 `status` 更新为 `stale` 或 `deprecated`。
- 不得记录账号、密钥、token、cookie 或真实隐私数据。

## 记忆状态

| 状态 | 含义 | 使用规则 |
|---|---|---|
| `current` | 已确认的当前事实 | 可作为实现参考，但高风险变更仍需验证 |
| `draft` | 方向明确但细节未完整确认 | 使用前需要确认关键细节 |
| `assumption` | Agent 推进任务时形成的假设 | 不得作为最终事实依据 |
| `deprecated` | 已废弃或被新事实替代 | 只用于回溯 |
| `stale` | 疑似过期或与现状冲突 | 使用前必须重新验证 |

## 推荐写入来源

- 用户明确确认的业务规则、验收口径或项目偏好。
- 已合并代码对应的实现约束、模块边界或验证命令。
- 任务报告中的历史问题、根因、防复发规则和回归验证。
- ADR 或决策记录中的架构取舍。
- 旧文档索引中经过验证并迁移为当前事实的内容。
