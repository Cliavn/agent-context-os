# Agent Context OS 协作入口

本仓库是一个通用蓝图项目。任何 Agent 在修改本仓库前，必须遵守以下规则。

## 核心定位

- 本仓库只沉淀通用方法、模板和示例。
- 核心蓝图不得绑定具体项目、行业、技术栈或私有业务事实。
- 具体项目内容只能放在 `examples/`，且必须明确标注为示例。

## 必读顺序

1. `README.md`
2. `docs/00-system-overview.md`
3. `docs/01-context-routing.md`
4. 与本次修改相关的专题文档或模板

## 修改规则

- 修改核心蓝图时，必须保持通用性。
- 新增模板时，必须使用占位符，例如 `<项目名>`、`<模块名>`、`<业务对象>`、`<验证命令>`。
- 不得把账号、密钥、token、cookie、真实业务敏感数据写入文档或示例。
- 不得把某个项目的临时习惯提升为通用规则，除非已经抽象成可复用原则。
- 修改结构后必须运行或更新 `scripts/check-agent-context-os.ps1`。

## 文档风格

- 面向中文开发者，默认使用中文说明。
- 项目名、文件名、命令、协议和通用技术术语可以保留英文。
- 文档应短而可执行，优先写边界、流程、模板和验收标准。
